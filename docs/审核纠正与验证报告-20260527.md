## 1. 关键更正:挂载层缓存幻影

复核初期,Linux 挂载视图(`mcp__workspace__bash`)报告了如下"异常":

- `.git/index.lock` 存在且已 3.4 小时、无法删除;
- 128 个文件被修改,其中 recipe 子系统三文件(`registry.js` / `run-recipe.js` / `routes/recipe-routes.js`)语法损坏、被截断;
- 一大批文件呈现行尾(EOL)噪声。

**经 Windows 原生侧核实,以上全部为假象:**

- Windows 侧 `.git/index.lock` **不存在**(锁已释放);
- `git status` 工作树 **0 个改动**(干净);
- 三个 recipe 文件 `node --check` **全部正常解析**;
- `git checkout` 在 Windows 侧正常、无锁阻挡。

根因:Linux 挂载桥对该仓库 `.git` 与部分文件内容做了**陈旧缓存**,呈现出过期的锁、过期的"脏"状态与被截断的文件内容。这与本项目历史上"package.json 第 32 行被截断"属同一类挂载读取假象。

> 教训:凡涉及 git 状态/文件完整性的判断,一律以 Windows 原生侧为准。
