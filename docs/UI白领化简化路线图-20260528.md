# UI 白领化简化路线图

> 目标:让没有任何计算机/编程背景的办公型用户,**上手就能用**。
> 每个区域标了「技术度 1-5」(5 最极客),按"先改最痛"排序。
> 这是建议清单,你**勾选要做的**,下一批我照做并重编。

## 已完成 ✅

- **头部模式切换**:三个开关 → 单个下拉框「模式·执行 ▾」(b75c137)
- **工作区切换**:输入路径 → 「📂 选择文件夹」系统对话框 + 预检失败明示(ad70e65 / 27e281c)
- **下载安装包**:头部 📦 按钮直接打开 bundle 目录(ad70e65)
- **优化提示**:无文字时给醒目黄底提示而不是静默禁用(27e281c)
- **可视化面板**:5 个模板按钮(柱/线/饼/表/指标卡)+ JSON 实时校验 + 缩放/全屏/下载/复制链接(c3809e2)
- **可观测**:适配真实事件形态,token 用量与工具原因不再"未记录"(440b05c)
- **记忆面板**:全面白领化文案(术语→名词解释、键→名称、依据折到「更多」、保存→让我记住、删除→忘掉)(本批)
- **运行时缺失项**:有 sourceUrl 的缺失组件出现「📥 下载安装」按钮直接打开浏览器下载页(本批)

## 待办按优先级排

### P0 — 普通用户必碰、最容易劝退

| 区域 | 技术度 | 现状痛点 | 建议改法 |
| --- | --- | --- | --- |
| **设置 / API 设置**(ApiSettings) | 5 | 让用户填 Base URL、API Key、Provider、Model id —— 普通办公用户连"什么是 base url"都不知道 | 默认隐藏全部技术字段;给「我有 Kimi 账号 → 一键登录」+「我没有 → 怎么申请」两个明显入口。技术字段塞进折叠「高级设置」。 |
| **底部输入栏 模型选择**(Composer footer) | 4 | 下拉里有 Kimi/OpenAI/Claude/本地 4 个 provider,普通用户不知道差别 | 默认隐藏 provider,只显示当前模型名(可点切换);把"换 AI 提供商"挪到设置里。 |
| **新建对话上方"模板/@文件/#历史"提示** | 3 | 符号引导对非技术用户像加密 | 三个按钮(📝模板 / 📎引用文件 / 🕘历史)替代 slash 提示。 |
| **思考强度 选择**(快速/标准/深度) | 2 | 三个词没解释 | 鼠标悬停加说明:"快速=秒回但浅;深度=慢但仔细"。 |

### P1 — 高频用、轻度技术化

| 区域 | 技术度 | 现状痛点 | 建议改法 |
| --- | --- | --- | --- |

## 横向规则(所有面板共通)

1. **错误文案去技术化**:把 `Error: ECONNREFUSED 127.0.0.1:3017` 变成「Kimi 暂时连不上,稍后再试」。
2. **空状态写人话**:不是「No records」,而是「这里还没有东西。先去 XXX 做点什么」。
3. **每个面板顶部一句话说明它能帮你做什么**(像我刚给记忆面板加的那段)。
4. **隐藏"技术开关"折成「高级」展开**:`autoApprove`, `trustedRoot`, `verbatim`, `idempotencyKey` 等。
5. **统一图标语义**:📁 工作区、📦 安装、📥 下载、📌 记忆、⚙ 设置、🤖 AI、💬 对话。
