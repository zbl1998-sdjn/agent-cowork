# 实时性与运行时增强规划 — 20260528

> 用户反馈三个产品级缺口:
> 1. **Kimi 还是基于训练知识回答**,不知道现在是 2026 年
> 2. **解码还有问题**(具体表现待补)
> 3. **内置的 Python 库不全面**,数据分析跑不起来
>
> 本文做诊断、对照 Claude Cowork/Code 的做法、给改进方案与排期。

---

## 一、问题诊断与现状摸底

### 缺口 1:Kimi 不知道"今天是 2026 年" — 根因明确

**现状**(`apps/host/src/kimi/system-prompt.js`):

`buildSystemPrompt` 拼接的字段只有:能力描述、工具清单、Windows 环境提示、planMode/devMode 分支、skills 列表、memory text。**完全没有当前日期/时间、工作目录、操作系统版本、应用版本号、上次会话时间等环境信息**。

模型自然只能根据自己的训练截止时间(Kimi K2 大约 2024 年底)回答"今天是哪天"。它没有任何信号告诉它"现实世界的当前时间"。

**对照 Claude Cowork/Code**:
- 系统会在每轮请求注入 `<env>` 块,内含 `Today's date`、`Working directory`、`Platform`、`Model name`、`OS version` 等。Claude Code 的 system prompt 由 110+ 条字符串按环境/配置组装,日期会随每次启动重新计算。
- WebSearch 工具描述里还专门有一条:"The current month is May 2026. You MUST use this year when searching for recent information." — 把当前月份直接写进工具说明,逼模型用对年份。

### 缺口 2:实时信息能力 — 只有 WebFetch,没 WebSearch

**现状**(`apps/host/src/tools/builtin-tools.js`):

注册的网络相关工具:
- `web.fetch` — 给定 URL 抓内容(由 SSRF guard 过滤本地/内网)
- `SearchWorkspace` — 搜本地索引

**缺**:WebSearch / Bing / Google / 任意搜索引擎入口。模型即使被告知今天是 2026 年,也找不到"搜 AI 新闻"的入口,只能要么编、要么让用户自己提供 URL。这就是定时任务"明早搜 AI 新闻"实际跑起来也搜不到东西的原因。

**对照 Claude Cowork/Code**:
- 内置 `WebSearch` 工具,可指定 `allowed_domains`/`blocked_domains`、返回 markdown 链接、要求强制注明 Sources 段。
- 工具描述会强调"用当前月份/年份搜",避免模型用过时年份。

### 缺口 3:Python 沙箱只有 stdlib — 数据分析根本跑不起来

**现状**:

- `apps/windows-client/resources/python-embedded/` 是 Python 3.12.10 官方 embeddable zip(`prepare-embedded-python.ps1` 下载)。
- 该 zip **只含 stdlib + 空的 `Lib\site-packages`**,既没 pandas、没 numpy、也没 matplotlib / openpyxl / pillow / requests。
- `prepare-embedded-python.ps1` 只是解压 zip + 在 `python312._pth` 里加上 `Lib\site-packages` 这行(允许第三方包搜索),**完全没装任何 pip 包**。
- `sandbox/code-runner.js` 调 Python 时直接 spawn `python.exe -c <code>`,代码里 `import pandas as pd` 立即 `ModuleNotFoundError`。

**对照 Claude Cowork/Code**:
- 沙箱预装 pandas / numpy / scipy / scikit-learn / statsmodels / matplotlib / seaborn / pyarrow / openpyxl / xlsxwriter / pillow 等。运行环境 Ubuntu 24.04 + Python 3.12.3 + Node 18,9GB RAM / 5GB 磁盘。
- 没装的小众库可以 `pip install --break-system-packages` 临时装(慢,但能跑)。

### 缺口 4:解码 — 现状已有基础处理,需要 reproducer

**现状**(`apps/host/src/kimi/cli-runner.js` + `api-runner.js`):
- CLI stdout/stderr 解码:`new TextDecoder('utf-8', { fatal: true })` 先试,失败 fallback 到 `gb18030`,都失败再 fallback 到默认替换字符。
- 流式 SSE:`decoder.decode(value, { stream: true })` 处理跨 chunk 中文字节边界。
- spawn Python 时设了 `PYTHONIOENCODING: 'utf-8'`。

**怀疑方向**(需要用户给具体重现):
- 某些 PowerShell 输出走 GBK / 控制台 codepage,不是 UTF-8 也不是 GB18030 而是 CP936。
- Shell 工具拼接长输出时编码切换。
- Markdown / 工具结果里 emoji + 中文混排时 `String.length` vs `code unit` 在前端截断。

## 二、改进方案分批

### P0 — 三个 quick win,各一个 commit,1-2 天可完成

#### P0-A:System prompt 注入 env 块(立竿见影)

**做什么**:`buildSystemPrompt` 入参追加 `now` / `trustedRoot` / `osName` / `appVersion` / `provider+model`,渲染成顶部 `<env>` 块。

参考写法:
```
<env>
今天:2026-05-28(星期四)
工作目录:C:\Users\Administrator\Desktop\agent cowork
操作系统:Windows 11
应用版本:Agent Cowork v0.2.0
当前模型:Kimi K2(月之暗面 API)
</env>
```

在 system prompt 顶部,先于"你是 Agent Cowork…",这样模型即使被截断也能保留环境信息。

**改 5 处文件**:
- `apps/host/src/kimi/system-prompt.js` — 改 `buildSystemPrompt` 函数签名,加 env 块拼接
- `apps/host/src/kimi/agent/tool-loop.js` 或调用点 — 调用 `buildSystemPrompt` 时把 `now: new Date()` / `trustedRoot` / 应用元信息传进去
- `apps/host/src/kimi/system-prompt.test.js` — 测断言 env 块包含今日日期 + 工作目录
- system prompt 版本号 `SYSTEM_PROMPT_VERSION` bump `v1 → v2`
- `docs/agent-prompt-changelog.md`(若有)记录

**安全网**:单测覆盖"日期格式"、"工作目录被路径策略归一化"、"env 块出现在 prompt 顶部"。

#### P0-B:加 WebSearch 工具(让"搜 AI 新闻"真能跑)

**做什么**:在 `apps/host/src/tools/` 新建 `web-search.js`,选一个搜索后端:
- **方案 A — Bing Web Search API**(需要 Azure key,$3-7/千次)
- **方案 B — DuckDuckGo HTML 抓取**(免费、零 key,但有反爬,够个人用)
- **方案 C — Tavily / SerpAPI 等专业搜索 API**(SaaS、需付费但稳定)
- **方案 D — 让用户在 Settings 里选 + 填 key**(灵活,放在"API"tab 里加一行 Search Provider)

推荐 **D 兜底 B**:默认 DDG(零配置),Settings 里给 Bing/Tavily 入口让用户升级。SSRF guard 已现成,只过滤本地域名即可。

注册到 `builtin-tools.js`:
```js
{
  name: 'WebSearch',
  description: '联网搜索(返回 5-10 条结果,每条含标题/URL/摘要)。提供商可在 Settings 切换。',
  inputSchema: { query, allowedDomains?, maxResults? },
  handler: webSearchHandler,
}
```

System prompt 同步加一行:"需要查最新信息(新闻、文档版本、价格、是否还活着)就用 WebSearch,不要靠记忆。"

#### P0-C:Python 沙箱预装核心包(数据分析能跑起来)

**做什么**:扩展 `prepare-embedded-python.ps1`,在下载 embeddable + 放开 site-packages 之后,加一步 `pip install`:

```powershell
# 给 embedded python 装 pip(embeddable 默认没 pip)
& $pythonExe -m ensurepip --default-pip 2>&1 | Tee-Object ...
# 装基础数据包(锁定版本,sha-pin 以保证可重复 + 安全审计)
& $pythonExe -m pip install --no-cache-dir -r ./resources/python-embedded/requirements.lock
```

`requirements.lock`(初版):
```
pandas==2.2.3
numpy==2.1.3
matplotlib==3.9.2
openpyxl==3.1.5
python-docx==1.1.2
pillow==11.0.0
requests==2.32.3
beautifulsoup4==4.12.3
lxml==5.3.0
```

膨胀代价:预估 +300 MB 解压后大小;打包进 Tauri installer 后约 +80 MB 压缩。可接受(Claude Cowork 沙箱也 ~5GB)。

**降级方案**(若 installer 太大不想全打进去):
- 把 Python 包改成"启动时按需自下载"—— 第一次跑 Python 工具时,host 检测缺包就 spawn pip install 拉,缓存到 `.AgentCowork/python-packages/`。慢一次,后续即时。
- 在 `RuntimeDependenciesPanel` 里把"数据分析包"作为一个组件,缺失时显示📥下载安装(已有 UI 模式)。

#### P0-D:解码问题先要 reproducer

不动代码,在桌面里给用户一个"复现入口":发一条任意"中文 + emoji"的话给 Kimi,截屏。或者跑某个 Shell 命令看输出。**等用户给 1-2 个具体例子再下手**,避免空猜。

可能动手的方向:
- Shell 工具读 stdout 时强制 `PowerShell -OutputFormat Text -InputFormat Text` + 切码 `chcp 65001`(UTF-8 控制台)。
- 前端 markdown render 长串 emoji + 中文时检查 `Array.from(str)` vs `str.slice`。

### P1 — 增强 system prompt 表达力,1 周内做

#### P1-A:把 WebSearch / WebFetch 用法写进 system prompt

模仿 Claude Code 在 WebSearch 描述里 hard-code 当前月份的做法:
- WebSearch 工具描述里塞 `${当前月份年份}` 占位,host 每次构 schema 时填实际值
- system prompt 加一句:"用 WebSearch 时务必带年份(当前是 2026 年),搜'最新 AI 新闻'这种没年份的容易拉回 2023 年的旧结果"

#### P1-B:Skills 系统对齐 Claude

本仓库已有 skills 注入(`skills` 数组传 system prompt),但缺:
- 自动按用户 query 选 skill(目前只是被动列在 system prompt)
- skill 触发的反馈(用户看不到"这个回复用了哪个 skill")

可参考 Claude Cowork 的 Skill tool:每次有匹配 description 的 skill 时主动 `Skill(skill_name)` 调用,把 SKILL.md 内容动态注入。当前 system-prompt.js 只静态列了 skill name/description,不够。

#### P1-C:long_conversation_reminder

Claude Cowork 在长会话末尾会注入 long_conversation_reminder。本仓库无类似机制 — 长会话(50+ 轮)模型可能偏离 system prompt,可加同样机制。

---

### P2 — 中期能力升级,2-4 周

#### P2-A:Code execution tool 化(让 Python 不只是 fire-and-forget)

当前 `sandbox.run-code` 一发就完,无法迭代修错。Claude 的 code interpreter 可以:
- 跑出错 → 模型读 stderr → 改代码 → 重跑
- 文件读写(`Read`/`Write` 在 sandbox CWD)

参考 Claude 文档,把 `sandbox.run-code` 升级为可读 stderr / 可挂载工作区 / 可多轮的"代码会话"。

#### P2-B:Provider 增加 OpenAI 兼容 / Anthropic 直连

当前主要 Kimi。等模型知识截止跟不上时,加 Anthropic Claude 直连(就用 Claude Sonnet 4.6 / Opus 4.6 自身的实时知识 + tool use 能力,2026 年知识截止)。在 Settings 的"默认模型" tab 已经留好 provider 下拉,补 host 端 provider 路由即可。

#### P2-C:Tauri 自动加载 sidecar 时设置正确 CWD

我之前真机回归遇到的 desktop 启动时 sidecar 找不到 host.exe — 是因为 `Start-Process` 没设 `-WorkingDirectory`。Tauri 启动 sidecar 时也按相对路径走,build 后跑应该没问题,但开发时偶尔出错。把 sidecar 启动 wrapped 成 absolute path 解决。

## 四、规矩 #0 — 依赖图(P0 三个 quick win)

```
P0-A   system-prompt.js (L1)  ←  agent/tool-loop.js  ←  agent-runner.js (L2)
         ↓ deps: 无新增,改函数签名 + env 块拼接

P0-B   tools/web-search.js (L1) ──→ tools/ssrf-guard.js (现成)
         └─ register at  tools/builtin-tools.js (L1)
         └─ optional: routes/system-routes.js 加 Settings "search provider" 字段读写

P0-C   scripts/prepare-embedded-python.ps1  ──→  resources/python-embedded/requirements.lock (新)
         └─ build-time only, 不动 host 运行时
```

全部下行,无回环。安全网:三个改动各自补一两个单测/烟雾测;改完跑 `npm run check` 5 道门;桌面层走一次 `cargo tauri build --no-bundle --config '{"build":{"beforeBuildCommand":""}}'` 真机验。
