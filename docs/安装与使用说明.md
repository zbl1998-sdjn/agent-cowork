# Kimi Cowork 安装与使用说明

面向首次使用的用户。Kimi Cowork 是一个**本地办公智能体**:它能在你电脑上读写文件、运行代码、连接你的工具,所有操作都在本机完成,关键的写操作会先征得你同意。

> 适用范围:个人电脑本地使用。请勿把它暴露到公网或当作多人共享的服务器。

---

## 一、安装

1. 拿到安装包(任选其一):
   - `Kimi Cowork_0.1.0_x64-setup.exe` —— 双击安装(推荐,普通用户用这个)。
   - `Kimi Cowork_0.1.0_x64_en-US.msi` —— 企业/批量部署用。
2. 双击 `setup.exe`,按提示下一步完成安装。安装后桌面/开始菜单会出现 **Kimi Cowork** 图标。
3. 首次启动如果 Windows 安全中心(Defender/SmartScreen)弹出提示:这是因为安装包尚未做代码签名。确认来源可信后选择"仍要运行/更多信息→仍要运行"。
   - 正式分发前建议先做代码签名,届时不会再有此提示。

---

## 二、首次启动

1. 打开 **Kimi Cowork**,会先看到登录界面。两种方式:
   - **注册 / 登录**:用一个用户名 + 密码(至少 6 位)。账户只保存在本机,用于区分多人的数据。
   - **跳过登录(游客)**:点"跳过,先在本地使用"即可立即开始。游客数据同样保存在本机。
2. 进入主界面后,顶部能看到你的身份和"⚙ 设置"。

---

## 三、配置 API Key(让它能对话)

Kimi Cowork 需要一个 Kimi/Moonshot 的 API Key 才能和模型对话。

1. 点右上角 **⚙ 设置 → API**。
2. 在 **API Key** 填入你的 key(形如 `sk-...`),Base URL/模型一般保持默认即可。
3. 点 **保存**。

> 安全说明:你的 key 只会保存在本机的 `.KimiCowork/config.json`,**永远不会显示明文、也不会上传**。界面只显示"已配置"状态。换 key 时直接覆盖,清除时点"清除密钥"。

---

## 四、基本使用

- **对话**:在底部输入框直接说你的需求,例如"把这个文件夹里的 CSV 汇总成表格"。
- **斜杠命令**:输入 `/` 可快速调用命令(新建对话、设置、切换面板等)和任务模板。
- **@ 引用文件**:输入 `@` 可引用工作区里的文件。
- **文件改动需确认**:当它要写文件或运行命令时,会先弹出请求,你点同意才执行(可在顶部开"自动批准",但高风险操作仍逐次确认)。
- **产物**:它生成的文件会出现在"产物"面板,可"在系统中打开";图片/Markdown/PDF 也能在对话里直接预览。
- **多对话**:左侧栏可新建、搜索、置顶、导出、删除对话;登录用户的对话会按账户保存。
- **设置中心**:账户、外观(浅色/深色)、模型、API、以及"自检"(查看安全与运行状态)。

---

## 五、常见问题

- **一直转圈/没反应**:多半是没配置 API Key,或 key 无效。去"设置 → API"检查;"设置 → 自检"里 `api-key` 应为绿色。
- **提示"模型服务暂时不可用,已启用熔断保护"**:上游模型接口连续失败触发了保护,稍等片刻再试。
- **重启后看不到旧的注册账户/游客数据**:当前本地账户/会话在内存中,重启 host 后游客租户会重建。重要数据请及时导出(对话可"导出 Markdown")。
- **想换工作目录**:Kimi 只能在你授权的"工作区根目录"内读写,其外的文件会被拒绝(这是安全设计)。

---

## 六、安全与隐私

- 所有文件操作都被限制在你选择的**工作区根目录**内;指向目录之外的软链接/越权路径会被拒绝。
- 本机服务默认要求登录令牌才能调用接口,且不信任伪造的身份头。
- 上传会拒绝 `.exe/.bat/.ps1/.html/.svg` 等可执行/主动内容类型。
- 日志会自动按大小轮转,不会无限增长。
- 仅监听本机回环地址(127.0.0.1),不对外暴露。

如需把它用于多人/联网场景,请联系维护者——那需要额外的部署加固(关闭游客免登录、启用 JWT、代码签名等)。
