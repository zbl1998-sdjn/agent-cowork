### P0-T10 · host 渐进式 TypeScript 迁移(新增)
目标不是重写 host,而是在已拆成小模块的边界上加类型护栏,降低跨模块参数/返回值回归。

| 子项 | 任务 | 验收 |
| --- | --- | --- |
| T10a | 建 `checkJs` 基线:`allowJs/checkJs/noEmit` + JSDoc 标注关键公共面,先覆盖 opt-in 模块清单 | `npm run check:host-types` 可本地复现;发现真实类型问题但不要求一次吃完整个 host |
| T10b | 接入质量门禁:类型检查进入 `npm run check`/CI,允许短期白名单但必须写理由和到期条件 | 新增类型回归会红;白名单不吞掉新增文件 |
| T10c | 逐文件转 `.ts`:从纯函数、L0 安全/HTTP 工具、L1 叶子模块开始,每批不超过 3~5 个模块 | 运行行为不变;导入路径/打包路径清晰;host 测试进程隔离全绿 |
| T10d | 扩到路由/运行时/agent 核心前,先把对应接口类型和特征测试补齐 | 不用类型迁移顺手改架构或语义;每个转换 PR 都是小而可回滚 |

T10a/T10b 扩展记录(2026-05-25):Host 类型护栏从 19 个 opt-in 模块扩到 25 个,新增纳入 `artifacts/live-spec.js`、`artifacts/viz.js`、`artifacts/office-writers.js`、`connectors/oauth-permissions.js`、`storage/json-store.js`、`tools/tool-registry.js`,补齐对应 JSDoc 与 Node shim 的 `crypto.randomUUID`/`fs.renameSync`/`Buffer`/`zlib` 声明,并继续由 `npm run check:host-types`/`npm run check` no-emit 门禁看护。直接 `.ts` 转换仍需先明确当前直接 Node 运行模式下的导入/打包路径,不得为了迁移改语言栈或重写运行链路。

T10a/T10b 扩展记录(2026-05-26):继续按 P2 连接器线把 `connectors/oauth-github.js` 纳入 host opt-in `checkJs`,补齐 GitHub device flow 的 JSDoc 输入/输出、OAuth 动态错误字段与 fetch/json 边界类型;类型护栏覆盖增至 26 个模块。`npm run check:host-types` 与 OAuth connector 聚焦测试已通过,运行语义不变。
