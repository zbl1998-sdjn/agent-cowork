# 06 · 运行时依赖与集成安装(开箱即用)

> 目标:**非技术用户装完 Agent Cowork 就能用所有功能,全程不需要自己懂、自己装任何环境(Python、字体、浏览器等)。** 遵守 `00` 分层 / `01` 标准 / `04` 质量门禁;受决策 D1(v1.0 只锁 Windows)约束,只需处理 Windows。
>
> 三条原则:① **能捆就捆**(轻量、人人要用的,直接打进安装包,随装即用);② **重的按需下载**(体积大、非人人用的,首次用到时联网拉取,装包保持小);③ **装不了的探测+引导**(系统级/超大依赖,只检测是否存在,缺失就优雅降级并提示)。全部依赖**不污染系统 PATH/注册表**,只落在应用资源目录或 AppData。

---

## 背景:为什么需要这一份

应用本身用 JS/TS/Rust 写,**源码零 Python**;但 agent 在**运行时**会调用外部语言/工具干活(沙箱里跑 `python x.py`、文档技能、数据分析、OCR、网页自动化……)。这些能力隐含"用户机器上得有 Python / 字体 / 浏览器 / Tesseract"等依赖。非技术用户根本不知道这些是什么,**必须由安装流程替他们搞定**,否则相关功能会静默失败。本文把这些依赖一次性盘清、定好集成策略。

---

## A · 捆绑进安装包(随装即用,轻量、人人要用)

## C · 装不了 / 不该捆 —— 探测 + 引导(不阻塞主线)
| ID | 依赖 | 用途 | 策略 | 验收 | 估 |
| --- | --- | --- | --- | --- | --- |
| C1 | **WSL2 / Docker** | 真·网络隔离沙箱的 VM 后端(本地子进程后端无法真正断网) | 不捆绑(系统级功能/超大安装):启动探测,有则默认 VM `--network=none`;无则回退本地并**明确提示"本地不隔离网络"**(对接 `02` P2-A) | 有 WSL/Docker 时真隔离;无则有清晰提示,不静默 | M |

## D · 通用要求(贯穿 A/B/C)
- **【必做·确定要做】统一"依赖管理器"模块 + UI 面板**:集中负责 探测→显示状态→一键安装/下载→校验→版本锁定→升级;给用户一个"运行时/依赖"面板,一眼看到装了什么、缺什么、能否一键补齐(把"黑箱依赖"变成可见可控)。它和 E 段安装时的勾选是同一套组件清单的两个入口。
- **【必做·确定要做】优雅降级(呼应"永不空交代")**:任一依赖缺失时,相关技能要**明确告诉用户"缺 X、是否现在安装"**(用中文、纯人话),而不是静默失败或丢一段看不懂的报错。
- **【必做】安装包全程中文显示**:installer 界面、组件勾选、说明文案、进度/错误提示一律用中文(面向中文非技术用户);这是面向用户的第一印象,不能出现英文原文或乱码。
- **安全**:所有按需下载必须校验来源 + 哈希/签名;不写系统 PATH、不改注册表;一律落在应用资源目录或 AppData,卸载时可清理。
- **许可证**:随包重分发的第三方(Python、CJK 字体、各库、pandoc、tesseract 等)均为宽松许可,保留各自 LICENSE 声明即可。

## 排期建议
- **先做 A1/A2/A3/A5**(开箱即用的根基:界面渲染 + Python 文档能力 + 中文字体 + 原生运行库)——这几项直接决定"非技术用户装完能不能用文档类功能",优先级最高。
- **A4** 随 Node 沙箱能力一起。
- **B 段**跟随各自上层特性落地(B1 随 `05` A4、B2 随 `05` A6、B3 随 `05` A2c)。
- **C1** 跟随 `02` P2-A(VM 隔离)。
- D 段的"依赖管理器 + UI 面板"建议在 A 段落地后尽早做,作为后续所有依赖的统一入口。

## E · 安装时让用户自己选(已定的决策)
> 决策:**不替用户硬定"装哪些",而是在安装/首次运行时用提示让用户自己勾选;我们的责任是把每一项介绍清楚(干什么用、多大、不装会缺什么)。** 这样既不强迫所有人吞下一个大包,也不让小白漏装该用的东西。

**A 段(核心,默认装、不可取消)**:WebView2、内嵌 Python+轻量文档库、CJK 字体、Node、VC++ —— 这些是"开箱能用文档功能"的根基,不作为可选项,直接装。

**B 段(可选组件,带说明的勾选项)**:安装时(及事后在依赖面板里)以**纯人话 + 体积估算**呈现,用户按需勾。例如:

| 可选组件 | 给用户看的一句话介绍 | 约体积 |
| --- | --- | --- |
| 数据分析包 | 让 AI 帮你分析表格、算统计、出图表和报告 | ~200MB |
| 网页自动化 | 让 AI 帮你上网操作、自动抓取网页信息 | ~150MB |
| 文字识别(OCR) | 从图片 / 扫描件里把文字提取出来,支持中文 | 中等 |
| 文档格式转换 | Word / PDF / Markdown 之间互相转换 | 小 |
| 开发者工具(Git) | 代码与版本管理操作(面向开发者) | 小 |

**安装方式两选一(也用提示让用户选)**:① **现在全装**——适合没网络或想一次装好的用户;② **用到时再下载**——安装包小,首次用到某功能时按提示下载。两种都给清晰说明,默认推荐②。

> 关键:install 时的"勾选"不是一锤定音——通过 D 的依赖面板,用户**事后随时能加装/卸载**任一可选组件。install 提示和依赖面板是同一套组件清单的两个入口。

> 剩余的小决策(不阻塞、可后续定):按需下载**走官方源还是自建镜像**(自建更稳但要维护;先默认官方源,后续视稳定性再说)。

## F · 安装与生命周期细节(易漏的硬细节)
> 这些不是"装哪个依赖",而是"在真实 Windows 上把安装这件事做对"——很多是非技术用户/中文环境的经典翻车点。

| ID | 细节 | 为什么重要 / 场景 | 要求 | 估 |
| --- | --- | --- | --- | --- |
| F1 | **代理支持** | 中国网络下连 Kimi/Moonshot、按需下载组件常需走代理 | app 与下载器都尊重系统代理,并允许手填代理;无代理时给清晰失败提示 | M |
| F2 | **中文用户名/路径 + 长路径** | `C:\Users\张三\…` 非 ASCII + Windows 260 字符限制(Python site-packages 易超) | 全程 UTF-8;启用长路径支持;**专门用中文用户名机器测一遍**安装与跑任务 | M |
| F3 | **SQLite 原生绑定打包** | host 用 sqlite(记忆/鉴权/适配器),native 绑定没打进 SEA 就一跑就崩 | 确认 sqlite 绑定正确进包(或用 Node 内建 sqlite);干净机器验证 | M |
| F4 | **卸载干净移除** | 按需下载的 Chromium/Tesseract/数据科学包可能数 GB,残留=孤儿文件 | 卸载能清掉已下载组件 + 缓存;给"保留/删除我的对话与数据"选择 | S |
| F5 | **按用户安装(免管理员)** | 非技术用户:不弹 UAC、不需管理员体验顺滑得多 | 默认"仅当前用户安装";"全机安装"作为可选;明确二选一 | S |
| F6 | **更新时保留已下载组件/数据** | 升级 app 别把用户已下的几百 MB 组件和数据冲掉重下 | 更新流程保留 AppData 里的组件/venv/用户数据(接 `02` P4-B) | S |
| F7 | **首次运行引导(区别于安装器)** | 装完第一次打开要能上手:API key、选工作区、依赖体检 | 首启引导页:配 Kimi key、选工作区文件夹、跑一次依赖体检;全程中文 | M |
| F8 | **SmartScreen/杀软引导** | 自签名(D2)触发 SmartScreen 警告,非技术用户会怕 | 提供中文引导教用户如何通过;文档/页面图示(接 D2) | S |
| F9 | **磁盘空间预检** | 大组件几百 MB,空间不足装一半失败 | 安装/下载前查可用空间,不足时提前用中文提示 | S |
| F10 | **离线可用性** | 无网时不能整个不能用 | 核心本地功能(文件操作等)离线仍可用;仅模型调用/按需下载需要网,并给明确提示 | S |

> F 段里 **F1 代理 / F2 中文路径 / F3 SQLite 打包** 是优先级最高的三项——它们直接决定"在一台普通中文用户的机器上,装完到底跑不跑得起来",建议和 A 段核心捆绑一起验。
