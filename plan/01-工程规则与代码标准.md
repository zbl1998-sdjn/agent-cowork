# 01 · 工程规则与代码生成标准(强制)

> 每个任务在标记"完成"前必须逐条满足本文件。目的是:代码有美感、可维护、可扩展、永不退化成上帝类。

## A. 模块与文件
1. **单一职责**:一个文件只有一个"改动理由"。混了路由+IO+策略 = 拆。
2. **体积预算**:源文件软上限 **250 行**、硬上限 **400 行**;超硬上限必须拆或在白名单写明理由。函数软上限 ~50 行。
3. **依赖方向**:严格遵守 `00` 的分层;禁止反向 import、禁止环;领域模块间解耦,跨域协作用接口 + 注入。
4. **命名**:host 用具名导出(一个模块一个清晰公共面);文件名 = 职责(kebab-case);避免 `utils.js` 大杂烩。
5. **纯核 + 边缘 IO**:把可测的纯逻辑与 IO/副作用分离,核心函数无副作用、易单测。

## B. 可扩展性(为 v1.0 的多提供商/多连接器/多沙箱铺路)
6. **面向接口**:模型提供商、沙箱后端、连接器、工具都走统一接口 + 注册表(plugin 模式),新增 = 加一个实现文件,不改调用方。
7. **配置而非硬编码**:端口、模型、限额等集中到 config,不散落。
8. **契约稳定**:跨单元的 HTTP/IPC 形状改动要兼容或版本化,并更新 `lib/types`。

## C. 健壮性(把已踩的坑变成规则)
9. **永不空交代**:任何运行路径都要有用户可见结果——不空回复、不静默终止、错误可读且脱敏(沿用 no-reply 兜底)。
10. **写后校验**:生成/写文件后回读校验(挂载盘截断教训);长任务可取消、可"继续"。
11. **副作用先批准**:所有 mutating/高危工具经审批门禁;审批/进度/失败在 UI 清晰可见。

## D. 安全
12. 密钥永不日志/回显/入库明文(redaction);路径 jail 在工作区内;CORS 白名单;上传类型白名单。
13. 新增能力默认最小权限,显式开权而非默认放开。

## E. 测试与验收(严格门禁)
14. **每个新模块**配套聚焦单测;**每个 bugfix** 附回归测试;**每个特性**至少一次真实模型端到端实跑(留记录/截图)。
15. host/ui 测试套件必须在 `--test-isolation=process` 下全绿、可复现。
16. **重构前先有特征测试(characterization tests)锁定现有行为;重构过程中这些断言不改**——只验证行为不变,改坏立刻可发现、可回滚。
17. host 渐进式 TypeScript 迁移必须从 `checkJs` + JSDoc 开始;逐文件转 `.ts` 只做小批量、可回滚的类型化迁移,不得借机改语言栈、重写模块或改变运行语义。

## F. 提交与版本
16. 小而专的提交;Conventional Commits(`feat/fix/refactor/test/docs/chore(scope): …`);一个任务一组提交。
17. 里程碑打 tag + 出离线快照(git bundle)+ 归档安装包到 `releases/<ver>/`。

## G. "完成"清单(Definition of Done,逐条勾)
- [ ] 通过 `node --check` / `tsc` / 架构与文件体积看护脚本
- [ ] 若触碰 host 公共接口或已纳入迁移清单的模块,通过 host 类型护栏(`checkJs`/JSDoc/后续 `.ts`)
- [ ] 新增/回归测试已加且全绿(进程隔离下)
- [ ] 无文件超硬上限(或白名单写明理由)、无新增上帝类、无 import 环
- [ ] 无密钥/敏感信息泄漏;副作用有审批;错误有兜底
- [ ] 关键决策写了"为什么"的注释;更新了相关文档/CHANGELOG
- [ ] 验收标准达成(对应 `02` 里该任务的"验收")
