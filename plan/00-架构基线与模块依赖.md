# 00 · 架构基线与模块依赖(v1.0 地基)

> 这是"做任何新特性之前必须先立"的地基:三块可部署单元、分层依赖方向、禁止反向依赖、上帝类清单与拆分目标。后续每个任务都必须遵守本文件的依赖规则。

## 0. 三块可部署单元(物理边界)
| 单元 | 技术 | 职责 | 绝不做 |
| --- | --- | --- | --- |
| 桌面外壳 `apps/windows-client/src-tauri` | Rust/Tauri 2 | 开窗、原生拉起/守护 host sidecar、安全的 open_path、IPC | 不放业务逻辑(保持薄) |
| 本地主机 `apps/host` | Node(SEA 打包) | 真正的大脑:HTTP/SSE API、Agent 循环、工具、沙箱、记忆、存储、鉴权 | 不依赖前端、不依赖 Tauri |
| 前端 `apps/windows-client/ui` | React/TS | 展示与交互,经 HTTP/SSE 调 host;经 @tauri-apps/api 调外壳命令 | 不直接 import host 代码 |

跨单元只能通过**契约**通信:UI↔host 走 HTTP/SSE(`lib/api.ts` 是唯一出口);UI↔外壳走 Tauri invoke;外壳↔host 走"拉起进程 + 监听 3017"。

## 1. host 内部分层(依赖只能从外向内,禁止反向/环)
```
L0 基础(无内部依赖) : security/(path-policy, redaction) · http/(request-utils) · 纯工具函数
L1 领域(只依赖 L0)  : kimi/ · sandbox/ · memory/ · workspace/ · recipes/ · artifacts/ · mcp/ · tools/ · auth/
L2 运行时(依赖 L0/L1): runtime/(approvals, scheduler, circuit-breaker, runs-index, action-audit, hooks, subagent…)
L3 路由(依赖 L0-L2) : routes/(把单个 /api/* 处理收敛成模块)
L4 组装根(只此一层连线): server.js(创建依赖、挂载 routes、装中间件)+ main.js(进程入口)
```
**规则**:L_n 只能 import L_{<n};领域层之间尽量不互相 import(需要就抽到 L0 或经接口注入);严禁出现 import 环(P0 加一条"无环"测试/脚本看护)。

## 2. 前端内部分层
```
components/*  ← 只渲染 + 触发回调;不直接 fetch
hooks/*       ← (新增)封装数据/流/状态(useAgentStream / useConversations / useKimiInfo …)
lib/api/*     ← 唯一与 host 通信的传输层(分域:auth/chat/files/recipes/schedules/kimi-config/transport)
lib/types     ← 共享类型(单一事实来源)
App.tsx       ← 只做布局编排 + 组合 hooks/components(目标 < 250 行)
```
**规则**:组件不写 fetch;数据逻辑进 hooks;HTTP 细节只在 lib/api;类型只在 lib/types。

## 3. 桌面外壳(已基本达标,保持)
`config / sidecar / commands / security / error / lib` 六个小文件,职责单一,保持薄;新增原生能力 = 新模块 + 薄 command,不往 lib.rs 堆。

## 4. 上帝类清单(P0 必须拆,带目标行数)
| 文件 | 现状行数 | 问题 | 拆分目标 |
| --- | --- | --- | --- |
| `apps/host/src/server.js` | 1090 | 路由+CORS+鉴权+限流+静态+各 /api 处理全挤在组装根 | 中间件抽到 `http/middleware/*`,各 /api 抽到 `routes/*`,server.js 只做"建依赖+挂载",**< 200 行** |
| `apps/windows-client/ui/src/App.tsx` | 742 | 单组件管所有状态/副作用/渲染 | 抽 hooks + 子组件,App.tsx 只编排,**< 250 行** |
| `apps/windows-client/ui/src/lib/api.ts` | 745 | 一个文件包揽所有 host 调用 | 按域拆 `lib/api/{transport,auth,chat,files,recipes,schedules,kimiConfig}.ts`,**每个 < 200 行** |
| `apps/host/src/kimi/agent-runner.js` | 571 | 循环+工具组装+审批+收尾混在一起 | 抽 `tool-loop` / `tool-gating(approval)` / `finalize(no-reply 兜底)` / `toolset-builder`,**主循环 < 250 行** |
| `apps/host/src/memory/memory-store.js` | 490 | 读写+分层+检索耦合 | 抽 `store(IO)` / `layers` / `query`,**每个 < 250 行** |

> 注:`App.js / api.js / Composer.js` 等是 tsc 产物(已被 `.gitignore` 忽略),不是源文件,不计入上帝类;但应确保它们不进仓库、不被 vite 误当源。

## 5. 依赖看护(P0 落地)
- 加 `scripts/check-arch.mjs`:扫描 import 方向,违反 L 层规则或出现环 → 失败(进 `test:host` 前置)。
- 加 `scripts/check-filesize.mjs`:源文件超软上限(250 行)告警、超硬上限(400 行)失败,白名单需写明理由。
